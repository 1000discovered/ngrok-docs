# Actions

Traffic Policy actions enable you to modify the behavior of traffic flowing through your endpoints. The following actions are available for use on your endpoints:

| Type                               | Description                                                                              | Supported Phases      |
| ---------------------------------- | ---------------------------------------------------------------------------------------- | ----------------- |
| [add-headers](add-headers)         | Add headers from incoming requests or outgoing responses.                                | `on_http_request`, `on_http_response` |
| [custom-response](custom-response) | Send back a pre-defined custom response to the client.                                   | `on_http_request`, `on_http_response` |
| [deny](deny)                       | Reject incoming traffic to an endpoint.                                                  | `on_tcp_connect`, `on_http_request`           |
| [jwt-validation](jwt-validation)   | Validate JSON Web Tokens (JWT).                                                          | `on_http_request`           |
| [log](log)                         | Emit an event with custom defined metadata.                                              | `on_tcp_connect`, `on_http_request`, `on_http_response` |
| [rate-limit](rate-limit)           | Rate limit traffic to your upstream servers.                                             | `on_http_request`           |
| [remove-headers](remove-headers)   | Remove headers from incoming requests or outgoing responses.                             | `on_http_request`, `on_http_response` |
| [redirect](redirect)               | Transform your URL using regular expressions and redirect users through location header. | `on_http_request`           |
| [url-rewrite](url-rewrite)         | Transform request URLs using regex.                                                      | `on_http_request`           |
